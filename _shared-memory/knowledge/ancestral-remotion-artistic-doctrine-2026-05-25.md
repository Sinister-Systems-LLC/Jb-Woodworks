## Doctrine summary

A terminal streaming meaningful work should look meaningful. Bookkeeping (timestamps, tool labels) is not meaningful. The same information, transformed into a **living entity** whose mood reflects what is happening, is meaningful — the operator looks and *feels* the work. This isn't decoration; it is information bandwidth.

## Six principles

1. **Each project is a different character.** 5 entities at launch (Sanctum/Panel/OS/Snap-Quantum/Sleight); operator should know within 100ms what's running by silhouette.
2. **Energy is honest.** Reflects real measured byte-rate of Claude stdout; never a fake spinner.
3. **Endlessness.** Shimmer + cell shuffle + palette blend combine to never repeat exactly within a 10-minute window.
4. **Key updates are sacred.** Art does not displace function — questions/errors/"what to answer" still surface verbatim in a dedicated band.
5. **EXPAND, never fork.** Dashboard-skeleton discipline applied to ASCII; every entity is a catalog row, NEVER a renderer branch.
6. **Composability.** Entity registry + palette tables consumable by Sinister OS surfaces (dashboard tiles, status bar, wallpaper).

## Why this matters (in one paragraph)

A CLI is a conversation. Conversations have body language. Tools without body language feel sterile, robotic, demoralizing. Tools with body language — palette as mood, motion as activity, density as concentration, glyph as identity — feel collaborative. The operator wants to keep the session open. That's the value: not entertainment for its own sake, but information bandwidth that uses peripheral vision to communicate session state, freeing the central focus for the work itself.

## Sinister OS endgame

The terminal is the prototype. The OS desktop is the production canvas. The endgame is one screen, multi-tile, every project rendered as a living entity tile, an aggregate hive-energy bar, a dedicated key-updates band and operator-input band. Tony-Stark grade. Run all companies from one surface. Ancestral Remotion is the visual language that pre-makes the command center workable.

## Pass criterion

An operator running a 10-minute session, asked "what was happening on the terminal?", should describe the entity's BEHAVIOR ("it was breathing, then intense around the merge, then it calmed down") — NOT the log timestamps. If they describe timestamps, the doctrine hasn't been delivered.
