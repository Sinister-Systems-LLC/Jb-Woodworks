**Compiled:** 2026-05-23

## What's in this folder

This package contains marketing, SEO, and social-media playbooks for Show Masters Production Logistics (https://www.showmasters.com), plus a set of blog posts ready to publish on the site.

Everything is in markdown (`.md`) format. You can:

1. **Read directly** — markdown renders cleanly in any text editor, in VS Code, or in a markdown viewer
2. **Hand to a contractor** — these read like a brief, with specifics a contractor can execute against directly
3. **Reference as a strategy document** — each file's "Quick wins" section is meant to be actionable without needing approval from a marketing committee

## How to use this on day one

**If you have 30 minutes:** Read `01-SEO.md` first. The "Quick wins (do this month)" section at the top of each file is meant to be actionable today.

**If you have a marketing contractor or in-house lead:** Hand them this whole folder. The instructions are detailed enough they can scope a 90-day engagement directly from it.

**If you'd rather summarize each file before reading the full thing:** Drop any file into a chat-based AI tool (ChatGPT, Gemini, Claude) with a prompt like *"Read this. What three things should I do this month?"* and you'll get a usable answer.

## The blog posts

The 11 posts in `blog-posts/` are written in SMPL's voice — direct, no marketing fluff. They serve double duty:

- **SEO content** — each post targets a specific search query a producer or stagehand would type
- **Authority content** — substantive enough to forward to a producer as "here's how we think about X"
- **Recruitment content** — three of them are deliberately written for stagehands considering applying, addressing pay, career path, and what the work actually looks like

If you publish them at one per month, the calendar fills eleven months. The dates baked in match a 2026 calendar; tweak before publishing.

## Caveat

These playbooks were assembled fast as a starting draft. They are deliberately specific (real venue names, real cert types, real Orlando/Dallas addresses, real role taxonomy), but some of the specifics may not match SMPL's actual current operations exactly. Read them as a strong starting draft, not a final policy document. Run the numbers and operational details past someone inside the company before publishing externally.
