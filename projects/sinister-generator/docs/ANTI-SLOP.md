# ANTI-SLOP — visual review checklist

> **Author:** RKOJ-ELENO :: 2026-05-23

Before any generated image is saved to a project's `outputs/<slug>/<type>/` folder (i.e. promoted out of `_rejected/` or staging), it must pass this checklist. Reject and regenerate if any line fails.

## The checklist

### 1. Text fidelity

- [ ] No hallucinated text. The model often invents gibberish "letters" — reject any.
- [ ] If text was intentional (e.g. "JOKR" lettering), it's spelled correctly and legibly.
- [ ] If text wasn't requested, there is none.

### 2. Anatomy

- [ ] Hands have 5 fingers (4 + thumb), correctly proportioned. No fused, missing, or extra digits.
- [ ] Eyes match (size, direction, color). No cross-eye, no asymmetric pupils unless intentional.
- [ ] Limbs end where limbs end. No "boneless" arms or unexplained extra limbs.

### 3. Brand-lock conformance

- [ ] Palette matches the per-project `BRAND.md` swatch list.
- [ ] Character traits preserved (for JKOR: purple skin + horns + crown + jester staff + cards + cheeky grin).
- [ ] Background respects the brand's "less / more loud" calibration.

### 4. UI cruft

- [ ] No download icons, share icons, save icons, watermarks, or app-overlay buttons.
- [ ] No source-app captions ("PRIMARY WHIMSICAL...", "Generated by...", "Imagen output").
- [ ] No rounded glow border that wasn't asked for.

### 5. Composition

- [ ] Subject is at the focal point per the brief (left third / centered / etc.).
- [ ] No critical content cut off at the canvas edge unless intentional.
- [ ] Aspect ratio is close to the target (within ~10% — Gemini doesn't honor exact pixel dimensions).

### 6. Lighting & tone

- [ ] Lighting matches the brand mood (JKOR = stage-lit dark; JBW = warm directional; SMPL = volumetric stage).
- [ ] No "stock photo / daylight" feel sneaking into a moody brand.
- [ ] Contrast between subject and background is healthy (subject pops, background recedes).

### 7. Reusability

- [ ] Resolution is at least 1024×1024 (Gemini's default output). Acceptable for web. For print, regenerate at higher res or upscale.
- [ ] PNG is RGB (not RGBA with alpha matte issues).

## Reject decisions

If 1+ line fails: **move to `outputs/<slug>/_rejected/`** with a `.txt` note describing why. Don't just delete — failed variants help future prompt iteration.

```
outputs/jkor/_rejected/
├── banner-v1.png
├── banner-v1.png.meta.json
└── banner-v1.reject.txt    ← "lost cartoony jester face, character looks like sorcerer not jester"
```

## Operator visual review

Even after the agent passes the checklist, the **operator gets the final visual yes/no** before the asset goes into a project's deliverables. Especially for brand-critical surfaces (homepage hero, product banner, social profile).

## Brand-specific overrides

Per-project `BRAND.md` may add additional anti-slop rules. E.g. JB Woodworks: "no plastic / faux wood finishes." Showmasters: "no daylight stock-photo lighting." Those merge with this checklist — never replace it.
