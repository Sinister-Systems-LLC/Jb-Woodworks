## Visual identity

**Hero character** (carried over from the operator's base banner):
- Purple-skinned demon / jester hybrid
- Small horns, small gold crown
- Mischievous grin, expressive eyes
- Holds a fan of playing cards in one hand
- Holds a sorcerer's wand or staff with a star tip in the other
- Dressed in purple-and-gold royal-jester attire (cape / collar)
- Posed center, looking at viewer

**Background** (re-spec from operator's "less loud" feedback):
- Deep near-black base: `#0A0B1E` (matches Sinister Command Center dashboard)
- Subtle vertical Sanctum-purple glow behind the character: `#7A3DD4` → `#4B1F8B` fading to the dark base
- NO runic symbols, NO swirls, NO sparkles, NO magical-particle clutter
- Premium, clean, video-game UI quality — calm, not busy

**Anti-slop rules** (any generated image must pass before commit):
- No text in the image (typography goes in SVG / CSS overlay, not baked)
- No emojis
- No UI artifacts: no download buttons, no icons, no progress bars, no badges
- No busy patterns: no runes, no sparkles, no swirling particles
- No uncanny hands / fingers (regenerate if the model hallucinates them)
- No logos that aren't the operator's own brand

## How to generate

### Banner (one-click)

```cmd
generate-banner.bat
```

Drops `generated/banner-v1.png` + `generated/banner-v1.png.meta.json`.

### One-off ad-hoc

```cmd
python -m nano_banana ^
  --prompt "Half-body portrait of the JKOR demon-jester holding cards" ^
  --output "generated/portrait-cards.png" ^
  --brand jkor ^
  --ref "reference/00-base-banner-original.png"
```

The `--brand jkor` flag auto-appends the locked style suffix. `--ref` style-transfers from the reference image so the character stays consistent.

### From Python

```python
from nano_banana import jkor_image
result = jkor_image(
    prompt="The demon-jester throwing a card mid-air, motion blur on the card",
    output_path="generated/card-throw.png",
    ref_images=["reference/00-base-banner-original.png"],
)
print(result.status, result.output_path)
```
