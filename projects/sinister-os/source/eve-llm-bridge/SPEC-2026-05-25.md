## 1. Decisions (operator-canonical posture, no gate questions)

**Both surfaces are GO.** EVE-the-LLM is the operator's daily chat oracle; it must reach the desktop and the phone without the operator ever opening a browser.

| Surface | Verdict | When it ships |
|---|---|---|
| Sinister OS desktop status-bar pill + quick-chat overlay (Hyprland) | **YES** | P3 (EVE shell) — wired into `waybar` + `wofi`-style sheet |
| Sinister OS desktop CLI (`eve chat "..."`) | **YES** | P3, alongside the pill |

| Hetzner-direct fallback (curl panel endpoints) | **ALREADY WORKS** | No OS-lane action needed; documented for operator |

**Why both:** the operator already drives 14+ daily sessions through chat-style intents (panel utterances, agent pokes, NL tweaks). Forcing him to a browser tab breaks the "EVE-is-the-shell" thesis at exactly the moment he needs the LLM most (mid-flow tone tweaks, persona swaps, fast triage).

## 3. The local daemon (`sinister-eve-llm-bridge`)

### 3.1 Why a daemon at all (vs every surface calling Hetzner directly)

Five jobs only the local daemon can do:

1. **License-key custody** — the snap-chatter key lives at `/etc/sinister/secrets/snap-chatter.key` (`root:eve 0640`). Surfaces never see it.
2. **History cache** — local 200-msg ring buffer at `/var/lib/sinister/eve-llm/history.sqlite` so the pill renders the last 5 chats with zero network roundtrip.
3. **Persona cache + drift watcher** — caches `/api/chatter/groups`, polls every 60s, fires `dbus` signal when persona list changes (so the pill dropdown updates live).
4. **Offline gracefulness** — when Hetzner is unreachable, daemon returns `{error: "offline", retryAfter: 30}` instantly; surfaces show "EVE offline" badge instead of spinner-of-death.
5. **Audit log** — every `chat.send` lands in `/var/log/sinister/eve-llm.jsonl` for the operator to grep later.

### 3.2 Systemd unit

```
[Unit]
Description=Sinister EVE-LLM IPC bridge
After=network-online.target sinister-eve.service
Wants=network-online.target

[Service]
Type=notify
ExecStart=/usr/bin/sinister-eve-llm-bridge --socket /run/sinister/eve-llm.sock
User=eve
Group=eve
RuntimeDirectory=sinister
StateDirectory=sinister/eve-llm
LogsDirectory=sinister
ReadOnlyPaths=/etc/sinister/secrets
NoNewPrivileges=true
PrivateTmp=true
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 3.3 Implementation language

**Rust** (consistent with `source/eve-control/` choice for the EVE daemon proper). Single binary, ~400 LOC. Crates: `tokio`, `jsonrpc-core`, `reqwest`, `rusqlite`, `zbus` (for the dbus signal), `tracing`.

Prototype in Python first (`source/eve-llm-bridge/prototype/sinister-eve-llm-bridge.py`) so the operator can boot the CachyOS VM and curl-test before greenlighting Rust. Same pattern as `sinister-eve-mcp-bridge.py`.

## 4. JSON-RPC 2.0 contract (UDS, line-delimited)

Transport: UNIX domain socket at `/run/sinister/eve-llm.sock`, mode `0660`, group `eve`. Line-delimited JSON-RPC 2.0 frames (one request per line, one response per line). No HTTP layer — UDS is local, framing overhead is wasted.

Windows dev fallback (so the operator can dev-test on the current Windows workstation): TCP `127.0.0.1:7340`, loopback-only, no TLS, requires `SINISTER_EVE_LLM_DEV=1` env. Production Linux **never** opens TCP.

### 4.1 Methods

#### `chat.send`

Fire a chat completion. Proxies to `POST /api/chatter/test`.

```json
// request
{
  "jsonrpc": "2.0",
  "id": "<client-uuid>",
  "method": "chat.send",
  "params": {
    "persona": "default",         // or any group from persona.list
    "message": "...",
    "context": [                  // OPTIONAL — last N messages for thread continuity
      {"role": "user", "content": "..."},
      {"role": "assistant", "content": "..."}
    ],
    "timeout_ms": 30000           // OPTIONAL — default 30000
  }
}

// response (success)
{
  "jsonrpc": "2.0",
  "id": "<client-uuid>",
  "result": {
    "reply": "...",
    "persona_used": "default",
    "model": "claude-haiku-4-5-20251001",
    "tokens": {"in": 1234, "out": 567},
    "latency_ms": 1820,
    "panel_event_id": "evt_abc123"
  }
}

// response (error)
{
  "jsonrpc": "2.0",
  "id": "<client-uuid>",
  "error": {
    "code": -32001,
    "message": "panel_offline",
    "data": {"retry_after_s": 30, "last_success_ts": "2026-05-25T08:30:12Z"}
  }
}
```

Error codes: `-32001 panel_offline` · `-32002 license_invalid` · `-32003 rate_limited` (panel returned 429) · `-32004 timeout` · `-32099 unknown_upstream`.

#### `chat.history`

Return last N messages from the local ring buffer. No network call.

```json
{"jsonrpc":"2.0","id":"...","method":"chat.history","params":{"limit":5,"persona":"default"}}
```

Returns `[{ts_utc, persona, direction, content, panel_event_id}, ...]`.

#### `persona.list`

Return cached personas. Cache TTL 60s; force refresh with `{"force":true}`. Proxies to `GET /api/chatter/groups`.

```json
{"jsonrpc":"2.0","id":"...","method":"persona.list","params":{"force":false}}
```

Returns `{personas: [{id, label, description, accent}], cached_at_ts: "..."}`.

#### `persona.tweak`

Operator-driven NL persona edit. Proxies to `POST /api/chatter/tweak`. Subject to chatbot lane's 100/day Anthropic Haiku cap.

```json
{"jsonrpc":"2.0","id":"...","method":"persona.tweak","params":{"persona":"slack-tone","instruction":"shorter sentences, less formal"}}
```

Returns `{persona, new_prompt, diff, daily_remaining}`.

#### `events.subscribe` (streaming)

Subscribe to the chatbot lane's event stream. Single long-lived connection per client. Server emits server-sent-event-style frames: `{"jsonrpc":"2.0","method":"events.notify","params":{...}}`. Backed by `GET /api/chatter/events` on the panel (long-poll or websocket on the panel side; bridge re-emits as JSON-RPC notifications).

Used by the waybar pill to update the "last reply" preview live without polling.

### 4.2 What the bridge does NOT expose

- Raw panel HTTP — surfaces never need to know the URL.
- License key — never returned in any RPC.
- Other tenants' history (the panel is multi-tenant; bridge is single-tenant by operator design).
- Anthropic-API-key passthrough (the panel handles all upstream calls).

## 7. Security model

- **License key:** `/etc/sinister/secrets/snap-chatter.key` on Linux (`root:eve 0640`); Android Keystore on Pixel.
- **Bridge socket:** UDS `0660 root:eve`. Only members of group `eve` can connect (the operator's user is in `eve`).
- **TLS:** all upstream calls verify Hetzner cert; pinned to Let's Encrypt issuer (no system CA bypass).
- **Audit log:** `/var/log/sinister/eve-llm.jsonl`, rotated weekly by `logrotate`. Schema: `{ts_utc, method, persona, msg_hash, latency_ms, status}`. Message bodies are **hashed**, not stored, by default (operator can flip `audit_full=true` in `/etc/sinister/eve-llm.toml` for full-text capture during debug).
- **Rate limit local:** bridge enforces 30 req/min per UDS client (prevents a runaway script from burning panel quota).
- **No PII filter at bridge layer** — the operator's own infra; classifier sits on the panel side already.

## 8. Failure modes + behavior

| Failure | Bridge behavior | UI behavior |
|---|---|---|
| Hetzner unreachable | `-32001 panel_offline` returned <50ms | Pill icon goes red, tooltip says "EVE offline, last seen 8:30Z" |
| License invalid (panel 401) | `-32002 license_invalid` | Pill icon goes amber, click opens `/etc/sinister/secrets/snap-chatter.key` in editor (if `SINISTER_DEV=1`) or shows toast "operator action: rotate snap-chatter key" |
| Panel 429 rate-limited | `-32003 rate_limited` with `retry_after_s` | Toast "EVE busy, retrying in 30s"; auto-retries once on backoff |
| Bridge daemon dead | UDS connect fails | Surface falls back to direct HTTPS to panel (degraded mode, no history cache) |
| Local ring-buffer corrupt | SQLite reinit, log warning | History pane shows "history reset", new chats work fine |
| `chat.send` timeout (>timeout_ms) | `-32004 timeout` | Spinner cancels, message stays in input box for retry |
