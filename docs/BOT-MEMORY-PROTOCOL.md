# Bot Memory Protocol - how Sinister Bots absorb + grow

**Hard constraint:** No bot self-modifies its system prompt silently. Every fact
added to a bot's persistent memory goes through an explicit `<bot>.absorb(fact, source)`
MCP call that **logs to an append-only audit ledger**. The operator can review,
revert, or pin any absorbed fact at any time.

## File layout

```
12_LLM_ORCHESTRATION/
  BOT-MEMORY-PROTOCOL.md                this file
  agents/
    _shared/
      bot_memory.py                     loader + absorb impl (Tier 2 / Tier 3 bots import this)
      __init__.py
    <bot>/
      server.py                         MCP server (already exists)
      SYSTEM-PROMPT.md                  bot's CANONICAL system prompt (hand-edited; absorption-aware)
      learned.md                        bot's PERSISTED absorbed facts (operator-reviewable; one fact per line)
      KNOWN-GOTCHAS.md                  bot-specific gotchas pulled from 09_REFERENCE/SANDBOX-GOTCHAS.md
  runtime-state/
    absorption-log.jsonl                append-only audit ledger of every absorb() call
    token-usage.jsonl                   existing usage log (unchanged)
09_REFERENCE/
  SANDBOX-GOTCHAS.md                    UNIVERSAL gotcha doc (regenerated by 08_AUTOMATIONS/aggregate-gotchas.ps1)
```

## Lifecycle of an absorbed fact

```
operator says "remember: TT Frida-spawn is sandbox-blocked, use pure-API task #26"
   |
   v
operator (or another bot) calls:
   researcher.absorb(
       fact="TT Frida-spawn is sandbox-blocked; use pure-API task #26 instead",
       source="operator session 2026-05-18, screenshot",
       tags=["gotcha", "tiktok", "frida"]
   )
   |
   v
bot_memory.absorb():
   1. validates: fact must be a string, source must be present
   2. dedupes: skip if same fact already in learned.md
   3. appends to agents/researcher/learned.md (atomic write):
      `- [2026-05-18T09:15Z] {fact} (source: {source}) [tags: gotcha,tiktok,frida]`
   4. logs to runtime-state/absorption-log.jsonl:
      `{"ts": "...", "bot": "researcher", "fact": "...", "source": "...", "tags": [...]}`
   5. returns {ok: true, count_after: <N>}
   |
   v
Next time researcher's system prompt is built, bot_memory.load() concatenates:
   - SYSTEM-PROMPT.md (hand-edited; canonical role)
   - 09_REFERENCE/SANDBOX-GOTCHAS.md (universal gotchas)
   - agents/researcher/KNOWN-GOTCHAS.md (bot-specific gotchas)
   - agents/researcher/learned.md (absorbed facts)
   |
   v
The Haiku call includes ALL of the above as the cached system prompt.
The bot now "knows" about the TT Frida deny and proposes the pure-API path.
```

## Why this design

**No silent drift.** Every learned fact has a timestamp, source, and audit-log
entry. The operator can `git diff agents/researcher/learned.md` (if they make
the hub a git repo) to see exactly what the bot learned over time. They can
delete a line at any time.

**No classifier evasion.** The gotcha doc is documentation OF the classifier's
behavior, not a way around it. Bots are taught "Path A is blocked, propose
Path B instead" - they don't try Path A and quietly mask the failure.

**Per-bot scope.** Each bot has its own learned.md. The translator doesn't see
researcher's facts (irrelevant + token waste). Cross-bot insights go in the
universal SANDBOX-GOTCHAS.md.

**Cache-friendly.** The combined system prompt (hand-edited + gotchas + learned)
is sent to Anthropic with `cache_control: ephemeral`. First call writes the
cache; subsequent calls within 5 min read it for ~1/10th the cost. So a bot
with 200 learned facts is still ~$0.02-$0.05 per call.

**Tier-1 bots opt out.** Pure-Python bots (sentinel, translator, watcher,
auditor, sinister-bus, custodian, stealth-browser) don't have LLMs to feed -
they don't import bot_memory at all. Tier-2 (triage with Ollama, librarian with
Ollama, researcher with Ollama) and Tier-3 (scribe with Haiku, curator with
Haiku) opt in.

## Operator controls

```
# In Claude session:
<bot>.absorb(fact, source, tags=[])      # add a fact
<bot>.list_facts(limit=20)               # see what's been absorbed
<bot>.forget(fact_id)                    # remove a specific fact (atomic; logged)
<bot>.export_memory()                    # dump learned.md content
<bot>.import_memory(path)                # bulk-import from a curated file

# Outside Claude:
edit agents/<bot>/learned.md directly    # hand-curated; loader picks it up next call
edit agents/<bot>/SYSTEM-PROMPT.md       # core role tweak
.\08_AUTOMATIONS\aggregate-gotchas.ps1   # rebuild SANDBOX-GOTCHAS.md from memory scan
```

## Audit

`runtime-state/absorption-log.jsonl` is append-only. Each entry:

```json
{"ts": "2026-05-18T09:15Z", "bot": "researcher", "action": "absorb", "fact": "...", "source": "...", "tags": [...], "fact_count_after": 17}
```

`forget` entries also log here with `"action": "forget"`. Nothing is ever
silently deleted from the log.

## What bots ALWAYS do

- **Emit a TL;DR block** at the end of every plan, proposal, digest, or long response.
  Format: `## TL;DR` + "**How we won:**" (1 line) + "**What you need to do:**" (1-3 short bullets).
  If nothing to do, write "Nothing — done." Plain language; no jargon.
  Operator directive 2026-05-18 PM.

## What bots NEVER do

- Silently modify their own SYSTEM-PROMPT.md (only operator hand-edits).
- Absorb facts from web scrapes without operator-explicit `researcher.absorb(...)`.
- Inherit other bots' learned.md (per-bot scope).
- Carry facts across reboots without operator visibility (every fact is in a flat .md file the operator can `cat`).
- Try to bypass a classifier deny - they propose the green path documented in SANDBOX-GOTCHAS.md instead.

## Initial population

After this protocol ships, the operator should:

1. Run `.\aggregate-gotchas.bat` once to seed `09_REFERENCE/SANDBOX-GOTCHAS.md`.
2. Add hand-curated entries below the `<!-- HAND-EDITED-BELOW -->` marker in that file.
3. Per-bot, add a `KNOWN-GOTCHAS.md` if there are bot-specific landmines.
4. From this point, every "remember: X" becomes a `<bot>.absorb(fact=X, source=...)` call.
